# Data Sources

This file is the source record for the input CSV files used by the project.

Not every CSV in `Data/` is a downloaded dataset. Some files were built by the project team to hold modeling assumptions, and most of the other CSV files in `Data/` are outputs generated by the code.

URLs in this file were checked on April 23, 2026.

## Input CSV files

| File | Type | Source | Notes |
| --- | --- | --- | --- |
| `Data/MDAD.csv` | Downloaded dataset | Mars Dust Activity Database public release: `https://battalio.com/mars-dust-activity-database/` | Dust storm catalog used to estimate storm duration and storm frequency near Jezero Crater. |
| `Data/nuclear_capacity_data_annual.csv` | Downloaded dataset | U.S. Energy Information Administration, Electric Power Annual Table 4.08.B: `https://www.eia.gov/electricity/annual/table.php?t=epa_04_08_b.html` | Annual U.S. nuclear capacity factor data used as the fission baseline. |
| `Data/fusion_reactor_specs.csv` | Project team table | Built from public reactor references: SPARC paper `https://doi.org/10.1017/S0022377820001257`, Princeton Satellite Systems page `https://psatellite.com/fusion-power-for-a-mission-to-mars/`, Avalanche Orbitron page `https://www.avalanchefusion.com/orbitron` | This file was assembled by the team. It was not downloaded as one ready made CSV. |
| `Data/technology_screening_inputs.csv` | Project team table | Built by the team as a transparent screening-rubric input table | Stores TRL and Mars operating penalty inputs used to compute the adapted McKelvey chart coordinates. The coordinates are generated by code from these inputs plus the LCOE model outputs. |
| `Data/habitat_engineering_constants.csv` | Project team table | Built from project assumptions, NASA Mars DRA 5.0 `https://ntrs.nasa.gov/archive/nasa/casi.ntrs.nasa.gov/20090040343.pdf`, and MOXIE background material `https://www.science.org/doi/10.1126/sciadv.abp8636` | Demand constants for life support, oxygen production, lighting, communication, and habitat thermal load. |
| `Data/storage_specs.csv` | Project team table | Built by the team as a simple storage benchmark table for the class model | This file was not downloaded from one external dataset. It stores assumed round trip efficiency, energy density, and cycle life values used in sensitivity work. |
| `Data/dust_penalty_curve.csv` | Project team table | Built by the team as a simple seasonal solar derating curve for the Mars year model | This file was not downloaded from one external dataset. It is a modeling simplification used to represent seasonal solar penalties. |

## Generated CSV files

The following CSV files are generated by the Python code when `run_analysis.py` is run:

- `Data/all_in_cost_model.csv`
- `Data/fission_scaling.csv`
- `Data/forecast_metrics.csv`
- `Data/lcoe_sensitivity.csv`
- `Data/mc_battery_sensitivity.csv`
- `Data/mc_sensitivity_grid.csv`
- `Data/mc_storm_sensitivity.csv`
- `Data/mckelvey_sensitivity.csv`
- `Data/monte_carlo_reliability.csv`
- `Data/resource_classification.csv`
- `Data/scenario_reliability.csv`
- `Data/solar_sizing_sensitivity.csv`

These generated files should be read as outputs of the project, not as outside source data.
